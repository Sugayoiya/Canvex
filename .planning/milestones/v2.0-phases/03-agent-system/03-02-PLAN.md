Create `api/app/agent/skill_toolset.py` implementing PydanticAI's `AbstractToolset` to bridge SkillRegistry → PydanticAI tools, per D1 (PydanticAI framework) and RESEARCH.md Pattern 1.

**Class: `SkillToolset(AbstractToolset)`**

Constructor parameters:
- `registry: SkillRegistry` — the singleton skill registry instance
- `context: SkillContext` — carries user/project/canvas identity for billing/logging
- `categories: list[str] | None = None` — optional category filter (e.g., `["TEXT", "EXTRACT"]`)
- `max_result_chars: int = 2000` — truncate tool results to prevent token explosion (Pitfall 5)

**`_build_tool_map()` method:**
1. Call `self._registry.discover(category=None)` to get all descriptors
2. Filter by `self._categories` if provided
3. Build `self._tools: dict[str, SkillDescriptor]` mapping safe_name → descriptor
4. safe_name conversion uses **category__skill namespacing** (double underscore) to prevent collisions (addresses review concern #6): `f"{desc.category.value.lower()}__{desc.name.split('.')[-1]}"` (e.g., `text.llm_generate` → `text__llm_generate`, `extract.characters` → `extract__characters`). Also store reverse mapping `self._original_names: dict[str, str]` from safe_name → original dotted name for invocation.
5. Check for duplicate safe_names — raise ValueError if found (Pitfall 4). Log warning with both colliding skill names for debuggability.

**`async def tool_defs(self) -> list[ToolDefinition]` method:**
Return list of `ToolDefinition` objects:
```python
ToolDefinition(
    name=safe_name,
    description=desc.description,
    parameters_json_schema=desc.input_schema,
)
```
Import from: `from pydantic_ai.tools import ToolDefinition`

**`async def call_tool(self, tool_call_id: str, name: str, args: dict) -> str` method:**
1. Look up descriptor from `self._tools[name]`, resolve original skill name via `self._original_names[name]`
2. If descriptor.execution_mode == "sync": directly `await self._registry.invoke(original_name, args, self._context)`
3. If descriptor.execution_mode == "async_celery": **cancellation-aware poll with jitter/backoff** (addresses review concern #4):
   - Call `registry.invoke()` to get task_id
   - Use exponential backoff with jitter: start at 1s, multiply by 1.5 per iteration, cap at 10s, add random jitter ±20% (`interval = min(base * 1.5**i, 10) * random.uniform(0.8, 1.2)`)
   - Per-skill timeout override: `timeout = getattr(desc, 'timeout', 120)` — default 120s, but individual skills can override via descriptor
   - **Cancellation-aware**: check `asyncio.current_task().cancelled()` and respect caller cancellation (when SSE client disconnects, the asyncio task is cancelled, breaking the poll loop cleanly)
   - Loop: `await asyncio.sleep(interval)` then `registry.poll(task_id)` 
   - Break when status is "completed" or "failed"
   - On timeout → return JSON error `{"error": "Tool execution timed out", "task_id": task_id}`
   - On `asyncio.CancelledError` → re-raise (let caller handle disconnect cleanup)
4. Truncate result JSON to `max_result_chars` chars. For structured JSON outputs, prefer truncating the `data` field while preserving the outer structure (don't cut mid-JSON)
5. Return `json.dumps(result.to_dict(), ensure_ascii=False)`
6. On exception (except CancelledError): return JSON error string (never raise — PydanticAI handles tool errors via return value)

**`_cancelled` property:** Add `self._cancelled: bool = False` flag and `cancel()` method that sets it. The poll loop checks `self._cancelled` each iteration — this enables SSE endpoint to propagate client disconnect to active tool execution.

**`async def tool_names(self) -> list[str]` method:**
Return sorted list of tool safe_names.

**Important anti-patterns to avoid per RESEARCH.md:**
- Do NOT hardcode tools with `@agent.tool` decorators
- Do NOT pass raw SkillResult objects — always `json.dumps(result.to_dict())`
- Do NOT bypass SkillExecutor — route through `SkillRegistry.invoke()` which preserves the logging chain

**`AgentService` class with methods:**

`create_agent(provider: str, model_name: str) -> Agent[AgentDeps, str]`:
- Resolve PydanticAI model via `create_pydantic_model(provider, model_name)`
- Build system prompt via `build_system_prompt()` from context_builder
- Return `Agent(model, deps_type=AgentDeps, instructions=system_prompt, retries=1)`

`async create_session(db, user_id, project_id, canvas_id, title, model_name, provider) -> AgentSession`:
- Create new AgentSession record, commit, return

`async get_session(db, session_id, user_id) -> AgentSession | None`:
- Query by session_id + user_id, return or None

`async list_sessions(db, user_id, project_id, canvas_id) -> list[AgentSession]`:
- Query sessions filtered by user_id + project_id + optional canvas_id, ordered by updated_at DESC, limit 50

`async delete_session(db, session_id, user_id) -> bool`:
- Delete session by ID if owned by user

`async load_message_history(db, session_id, max_messages: int = 20) -> list`:
- Query most recent `max_messages` AgentMessage rows for session
- Find the LAST message with `pydantic_ai_messages_json` not null
- Deserialize via `ModelMessagesTypeAdapter.validate_json()`
- Per D4: simple truncation (keep recent N), per Pitfall 2: pin PydanticAI version range

`async save_messages(db, session_id, pydantic_ai_messages_json: bytes, user_content: str, assistant_content: str, tool_calls: list, tool_results: list, input_tokens: int, output_tokens: int)`:
- Save user message (role="user", content=user_content)
- Save assistant message (role="assistant", content=assistant_content, pydantic_ai_messages_json=pydantic_ai_messages_json decoded as str)
- Save tool-call/tool-result messages if present
- Update session.message_count and session.total_tokens
- Commit all in one transaction
