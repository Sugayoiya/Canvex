**1. Add `loadSessionHistory` action with stale-response guard to chat-store.ts:**

**Stale-response guard (review concern #5):** When user rapidly switches sessions (A → B → C), the getMessages response for session A may arrive after session C is already active. Without a guard, session A's messages would overwrite session C's messages. Fix: use a monotonically increasing `_loadRequestId` counter. Each `loadSessionHistory` call increments the counter and captures its value. When the response arrives, it checks if the current counter still matches — if not, the response is stale and discarded.

Add a `_loadRequestId` property to the store (internal, not in ChatState interface since it's implementation detail — use a module-level `let`):

At the top of the file, add:
```typescript
let _loadRequestId = 0;
```

In the `ChatState` interface, add:
```typescript
loadSessionHistory: (sessionId: string) => Promise<void>;
isLoadingHistory: boolean;
```

In the initial state inside `create`, add:
```typescript
isLoadingHistory: false,
```

In the `create` body, add after `setSession`:
```typescript
loadSessionHistory: async (sessionId: string) => {
  const requestId = ++_loadRequestId;
  set({ isLoadingHistory: true });
  try {
    const { agentApi } = await import("@/lib/api");
    const res = await agentApi.getMessages(sessionId, 50);

    // STALE RESPONSE GUARD: if user switched sessions during the fetch,
    // _loadRequestId will have been incremented again — discard this response
    if (requestId !== _loadRequestId) return;

    const rawMessages: Array<{
      id: string;
      role: string;
      content: string | null;
      created_at: string;
    }> = res.data?.messages ?? [];

    const messages: AgentMessage[] = rawMessages
      .filter((m) => m.role === "user" || m.role === "assistant")
      .map((m) => ({
        id: m.id,
        role: m.role as "user" | "assistant",
        content: m.content ?? "",
        timestamp: new Date(m.created_at).getTime(),
      }));

    // Double-check: only apply if still on the same session
    if (requestId !== _loadRequestId) return;
    set({ messages, isLoadingHistory: false });
  } catch {
    if (requestId === _loadRequestId) {
      set({ isLoadingHistory: false });
    }
  }
},
```

Note: Using dynamic import `await import("@/lib/api")` to avoid circular dependency risk since api.ts may import from stores.
