**Per D-01 and D-05, make these changes to `web/src/app/admin/page.tsx`:**

**1) Add imports:**
```typescript
import { useRouter } from "next/navigation";
```

**2) Add alerts interface and parallel query:**
```typescript
interface AlertsData {
  quota_warning_users: number;
  failed_tasks_24h: number;
  error_providers: number;
}
```

In `AdminDashboardPage`, add a second parallel query after the existing dashboard query:
```typescript
const router = useRouter();

const alertsQuery = useQuery({
  queryKey: ["admin", "alerts"],
  queryFn: () => adminApi.getAlerts().then((r) => r.data as AlertsData),
  refetchInterval: 60_000,  // matches dashboard refresh cadence — admin pages are low-traffic, 60s balances freshness vs server load
});
const alerts = alertsQuery.data;
```

**3) Dual-query state handling matrix (addresses review: state handling completeness):**

The dashboard page fires two independent queries. Handle all state combinations:

| dashboardQuery | alertsQuery | UI behavior |
|---|---|---|
| loading | loading | KPI cards show skeleton values, no alert badges |
| success | loading | KPI cards show values, no alert badges (alerts still loading) |
| success | success | KPI cards show values + alert badges where count > 0 |
| success | error | KPI cards show values, NO alert badges displayed (silent fail per UI-SPEC: "Dashboard alerts fetch error → silent fail") |
| error | any | Full error banner shown (existing pattern), alert badges irrelevant |
| loading | error | KPI skeletons shown, alert badges irrelevant |

Implementation: alert badges render ONLY when `alerts` is truthy (data loaded). If alertsQuery errors, `alerts` remains `undefined`, badges don't render, no error shown. This is the designed graceful degradation per UI-SPEC copywriting contract: "Dashboard alerts fetch error: Silent fail — KPI cards show without alert badges, no error shown."

**4) Extend KpiCard with `onClick`, hover effects, and keyboard accessibility:**

Add to the KpiCard destructured props:
```typescript
onClick?: () => void;
alertBadge?: string | null;
```

Add to the KpiCard root `<div>`:
- `onClick={onClick}` — navigation handler from parent scope
- `cursor: onClick ? "pointer" : "default"` — visual affordance
- `role={onClick ? "link" : undefined}` — accessibility (addresses review: keyboard/ARIA)
- `tabIndex={onClick ? 0 : undefined}` — focusable via Tab key
- `onKeyDown={(e) => { if (onClick && e.key === "Enter") onClick(); }}` — Enter key navigates
- `transition: "all 150ms ease"` — smooth hover animation
- `onMouseEnter`: if `onClick` defined, set `borderColor` to `"rgba(0,209,255,0.3)"` (--ob-glass-hover-border) and `transform` to `"translateY(-1px)"`
- `onMouseLeave`: reset `borderColor` to `"var(--cv4-border-subtle)"` and `transform` to `"none"`

**5) Add alert badge rendering inside KpiCard:**

{alertBadge && (
