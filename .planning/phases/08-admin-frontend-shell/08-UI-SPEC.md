# Phase 08 — UI Design Contract

> Visual and interaction contract for the Admin Frontend Shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Primary source of truth: `designs/admin-console-v2-neutral-gray.png`, `designs/component-specs.md`, `designs/design-tokens.json`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — project uses custom inline styles with CSS custom properties) |
| Preset | not applicable |
| Component library | none (custom components with inline styles, established Phase 06 pattern) |
| Icon library | lucide-react (already installed) |
| Display font | Space Grotesk (`var(--font-headline)`) |
| Body font | Manrope (`var(--font-body)`) |
| Token namespace | `--cv4-*` (reuse Canvas V4 tokens; admin values are 1:1 identical) |

**Implementation pattern:** Inline styles referencing `--cv4-*` CSS custom properties. This is the established pattern from Phases 04–06. No new CSS custom property namespace needed — admin shell reuses CV4 tokens directly since all design-tokens.json values map exactly to existing `--cv4-*` variables.

**One new token required:** `--cv4-btn-secondary-border` (dark: `#3c494e30`, light: `#D0D0D040`) — add to `globals.css` in both `:root` and `.theme-light` blocks.

### 60/30/10 Split

| Ratio | Elements | Dark | Light |
|-------|----------|------|-------|
| 60% dominant | Page bg, sidebar bg | `#131313` | `#F5F5F5` |
| 30% secondary | KPI cards, topbar, buttons | `#2A2A2A` | `#FFFFFF` |
| 10% accent | Primary button bg, active nav text | `#E5E2E1` | `#1A1A1A` |

**Accent reserved for:** Primary action buttons ("Generate Report"), active nav item text, page title text, KPI card values. NOT for decorative borders, link underlines, or brand color (no cyan `--ob-primary` in admin shell — per design decision §8.1).

**Destructive color:** Inherit `--ob-error` (`#FFB4AB`) from Obsidian Lens — used only for ban/disable confirmations in Phase 09+.

- Label: "Admin Console"

## Route Structure (CONTEXT D-04, D-05, D-06)

```
app/admin/
├── layout.tsx          ← AdminGuard + AdminShell wrapper
├── page.tsx            ← Dashboard (placeholder with KPI cards)
├── users/page.tsx      ← Placeholder
├── teams/page.tsx      ← Placeholder
├── quotas/page.tsx     ← Placeholder
├── pricing/page.tsx    ← Placeholder
├── providers/page.tsx  ← Placeholder
└── monitoring/page.tsx ← Placeholder
```

- Code splitting: Automatic via Next.js App Router — no `next/dynamic` needed (CONTEXT D-06)
- Admin bundle isolation: `app/admin/` only loaded when user visits `/admin/*`
- AdminGuard wraps in `layout.tsx`, re-validates `is_admin` via `/auth/me` on mount (REQ-17)
