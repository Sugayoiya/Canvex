# Phase 03 — UI Design Contract

> Visual and interaction contract for the Chat Sidebar, tool call display, and session management UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom components with TailwindCSS 4) |
| Preset | not applicable |
| Component library | none — hand-built with Tailwind utility classes |
| Icon library | lucide-react (already installed, v1.7.0) |

> Source: Codebase analysis — no `components.json`, no shadcn. Project uses TailwindCSS 4 with `@tailwindcss/postcss` plugin. All UI components are custom-built.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within message bubbles |
| sm | 8px | Inner padding for compact elements (badges, status pills) |
| md | 12px | Chat input padding, sidebar section gaps |
| lg | 16px | Message list item spacing, card padding |
| xl | 24px | Sidebar header/footer padding, section breaks |
| 2xl | 32px | Major section breaks (session list ↔ chat area) |

Exceptions:
- Chat sidebar width: **380px** collapsed toggle button + expanded panel
- Toggle button hit area: **40px × 40px** (accessible touch target)
- Chat input minimum height: **44px** (single line), max: **120px** (multi-line expand)

> Source: Existing patterns — canvas toolbar uses `px-3 py-2` (12px/8px), node cards use `p-3`/`p-5` (12px/20px). Scale aligned to observed usage.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Chat message text, tool result descriptions |
| Label | 12px | 500 (medium) | 1.4 | Timestamps, metadata, tool names, thinking status |
| Heading | 16px | 600 (semibold) | 1.3 | Sidebar title, session titles, section headers |
| Mono | 13px | 400 (regular) | 1.5 | Tool call arguments, JSON previews (Geist Mono) |

> Source: Existing codebase uses `text-xs` (12px), `text-sm` (14px), `text-xl` (20px), and semibold for headings. Chat UI needs compact density — 3 sizes plus mono for code.

## Color

### Light Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `gray-50` | Chat message area background |
| Secondary (30%) | `#f9fafb` / `gray-100` | Sidebar background, user message bubbles |
| Accent (10%) | `#4f46e5` / `indigo-600` | Send button, active session indicator, CTA links |
| Destructive | `#ef4444` / `red-500` | Abort button, error state badge |

### Dark Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` / `zinc-950` | Chat message area background (matches canvas bg) |
| Secondary (30%) | `#18181b` / `zinc-900` | Sidebar background, user message bubbles |
| Accent (10%) | `#6366f1` / `indigo-500` | Send button, active session indicator, CTA links |
| Destructive | `#ef4444` / `red-500` | Abort button, error state badge |

### Semantic Status Colors

| Status | Light | Dark | Used For |
|--------|-------|------|----------|
| Thinking | `amber-500` | `amber-400` | Thinking indicator dot pulse |
| Tool Call (in-progress) | `blue-500` | `blue-400` | Tool call spinner, "正在..." prefix |
| Tool Result (success) | `emerald-500` | `emerald-400` | ✅ checkmark, result summary |
| Tool Result (failed) | `red-500` | `red-400` | ❌ failure indicator |
| Streaming | `indigo-500` | `indigo-400` | Streaming cursor blink |

Accent reserved for: send button, active session highlight in session list, primary CTA links, streaming cursor indicator. **Never** for message text, borders, or background fills.

> Source: Auth layout uses `indigo-600` for primary actions. Canvas nodes use `cyan-400`/`cyan-500` for accents. Chat sidebar adopts `indigo` to match the app-level brand color, reserving `cyan` for canvas-specific elements.

## Component Inventory

| Property | Spec |
|----------|------|
| Position | Fixed right, full viewport height |
| Width (expanded) | 380px |

| Border | Left border: `border-l border-gray-200 dark:border-zinc-800` |
| Background | Secondary color (see Color section) |
| Layout | Flex column: header (fixed) → content area (scroll) → input (fixed bottom) |
| Two entry points | Project level (`canvas_id=null`) and Canvas level (`canvas_id=X`) |

### 2. `chat-session-list.tsx` — Session History

| Property | Spec |
|----------|------|
| Trigger | Click session list icon in sidebar header |
| Display | Dropdown overlay or slide-in panel from top of sidebar |
| Session item | Title (truncated 1 line) + last message preview + timestamp |
| Active session | Left accent border (`border-l-2 border-indigo-500`) + bold title |
| New session | "+" button at top, creates new session via API |
| Max visible | 8 sessions scrollable, sorted by `updated_at` desc |
| Empty state | Single-line text: "暂无历史会话" |

### 3. `chat-messages.tsx` — Message List

| Property | Spec |
|----------|------|
| Container | Flex column, `overflow-y-auto`, scroll-to-bottom on new messages |

| Assistant message | Left-aligned, no bubble (flush left), 14px body, full width |
| Tool call message | Collapsible block (see `tool-call-display.tsx`) |
| Tool result message | Inline status line below tool call (see `tool-call-display.tsx`) |
| Timestamp | 12px label, gray-400/gray-500, shown on hover or after 5-min gap |
| Auto-scroll | Scroll to bottom on new message; pause auto-scroll if user scrolls up |
| Loading | Skeleton placeholder (3 pulse lines) when loading history |

### 4. `chat-input.tsx` — Message Input

| Property | Spec |
|----------|------|
| Container | Fixed bottom of sidebar, horizontal flex |
| Input type | `<textarea>` with auto-resize (1→5 rows) |
| Placeholder | "向 AI 助手发送消息..." |
| Send button | Indigo accent, `SendHorizontal` icon (lucide), right of input |
| Abort button | Red destructive, `Square` icon (lucide), replaces send during streaming |
| Keyboard | `Enter` to send, `Shift+Enter` for newline |
| Disabled state | Input + send button disabled during streaming (abort button active) |
| Border | Top border: `border-t border-gray-200 dark:border-zinc-800` |
| Padding | 12px horizontal, 8px vertical |

### 5. `tool-call-display.tsx` — Tool Call Progress (Cursor-style)

| Property | Spec |
|----------|------|
| Layout | Vertical stack, indented 16px from left |
| Collapsed state | Single line: icon + tool name + status indicator |
| Expanded state | Tool name + arguments (mono font) + result summary |
| Default | Collapsed; click to expand/collapse |
| Status icons | Spinner (in-progress), ✅ (success), ❌ (failed) |
| Tool name display | Skill `display_name` (e.g., "创建剧本" not "script_generate") |
| Arguments preview | Truncated JSON in mono font, expandable |
| Result summary | 1-line summary text (e.g., "✅ 剧本已生成（1200字）") |
| Border | Left border: `border-l-2 border-blue-500/30 dark:border-blue-400/30` |
| Background | Subtle: `bg-gray-50/50 dark:bg-zinc-900/50` |
| Timing | Show elapsed time for tool calls > 3 seconds |

**SSE Event → Display Mapping** (from CONTEXT.md D7):

| SSE Event | Visual |
|-----------|--------|
| `thinking` | Fuzzy text: "正在分析你的需求…" / "正在规划接下来的步骤…" |

| `tool_result` | Result summary: "✅ 剧本已生成（1200字）" / "✅ 发现3个角色" |
| `token` | Streaming text output in assistant message area |
| `done` | Streaming cursor disappears, message finalized |
| `error` | Red error block with retry suggestion |

### 6. `thinking-indicator.tsx` — Agent Thinking Animation

| Property | Spec |
|----------|------|
| Layout | Inline within message list, left-aligned |
| Animation | 3 pulsing dots (`animate-pulse`), staggered delay (0ms, 150ms, 300ms) |
| Dot size | 6px circles |
| Dot color | `amber-500` (light) / `amber-400` (dark) |
| Label | 12px gray text next to dots: "正在思考..." or event-specific text |
| Duration | Visible from `thinking` event until `tool_call`/`token`/`done` event |
| Skeleton variant | For initial load: 3 horizontal bars at 60%/80%/40% width, pulse animation |

## Layout Integration

### Canvas Page Integration (`/canvas/[id]`)

```
┌─────────────────────────────────────────────────┬──────────┐
│                                                 │          │
│              Canvas Workspace                   │  Chat    │
│              (ReactFlow)                        │  Sidebar │
│                                                 │  380px   │
│                                                 │          │
│                                                 │          │
│                                                 │          │
└─────────────────────────────────────────────────┴──────────┘
                                                  ↑
                                              Toggle Button
                                              (when collapsed)
```

- Canvas workspace resizes with `calc(100vw - 380px)` when sidebar open, `100vw` when closed
- Transition: canvas width animates in sync with sidebar slide (`duration-200`)
- Toggle button: absolutely positioned at `right-0` of viewport when sidebar closed
- Sidebar renders as sibling of `CanvasWorkspace` inside the canvas page, NOT nested inside ReactFlow

### Project Page Integration (`/projects`)

- Same sidebar component, mounted at page level
- `canvas_id` prop = `null` for project-level agent context
- Toggle button in page header area (top-right corner)

## State Management

### Zustand Store: `chat-store.ts`

```typescript
interface ChatState {
  isOpen: boolean;
  sessionId: string | null;
  projectId: string | null;
  canvasId: string | null;
  messages: AgentMessage[];
  isStreaming: boolean;
  thinkingText: string | null;

  // Actions
  toggle: () => void;
  open: (projectId: string, canvasId?: string) => void;
  close: () => void;
  setSession: (sessionId: string) => void;
  addMessage: (message: AgentMessage) => void;
  updateLastMessage: (content: string) => void;
  setStreaming: (streaming: boolean) => void;
  setThinkingText: (text: string | null) => void;
  reset: () => void;
}
```

### React Query Keys

| Key | Endpoint | Refetch |
|-----|----------|---------|
| `["agent-sessions", projectId, canvasId]` | `GET /api/v1/agent/sessions` | On new session create |
| `["agent-messages", sessionId]` | `GET /api/v1/agent/sessions/{id}/messages` | On session switch |

### SSE Hook: `use-agent-chat.ts`

- Uses `@microsoft/fetch-event-source` for POST + auth headers
- AbortController for cancel/abort
- Updates Zustand store directly for real-time message state
- Falls back to polling if SSE connection fails 3 times

## Interaction States

### Sidebar States

| State | Visual | Trigger |
|-------|--------|---------|
| Closed | Hidden, toggle button visible at right edge | Default state |
| Open (empty) | Sidebar visible, empty state with suggestions | First open / no messages |
| Open (active session) | Sidebar visible, message history loaded | Select session or send message |
| Open (streaming) | Input disabled, abort button visible, messages streaming | After sending message |

### Message States

| State | Visual | Duration |
|-------|--------|----------|
| User sent | Right-aligned bubble, instant | Permanent |
| Thinking | Pulsing dots + thinking text | Until tool_call or token |
| Tool calling | Collapsible block with spinner | Until tool_result |
| Tool result | Inline result summary (✅/❌) | Permanent |
| Assistant streaming | Left-aligned text with blinking cursor | Until done event |
| Assistant complete | Left-aligned text, no cursor | Permanent |
| Error | Red-tinted block with retry button | Until retry or dismiss |

### Session List States

| State | Visual |
|-------|--------|
| Loading | Skeleton list (3 items) |
| Loaded | List of sessions with active highlight |
| Empty | "暂无历史会话" text |
| Creating | Optimistic new session at top with loading indicator |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard navigation | Tab through: toggle → session list → messages → input → send/abort |
| Focus trap | When sidebar is open, trap focus within sidebar panel |
| Screen reader | `role="complementary"` on sidebar, `aria-live="polite"` on message list |
| Reduced motion | Respect `prefers-reduced-motion`: skip dot pulse, use opacity fade instead |
| Color contrast | All text ≥ 4.5:1 ratio against backgrounds (WCAG AA) |
| Abort announce | `aria-label="停止生成"` on abort button |
| Live region | New messages announced via `aria-live="polite"` on message container |
