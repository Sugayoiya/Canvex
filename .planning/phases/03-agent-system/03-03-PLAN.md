**1. Create `api/app/api/v1/agent.py`** with agent chat and session management endpoints.

Use `router = APIRouter(prefix="/agent", tags=["agent"])` following existing pattern from canvas.py.

**Session CRUD endpoints:**

`POST /agent/sessions` — Create new session:
- Dependencies: `db = Depends(get_db)`, `user = Depends(get_current_user)`
- Request body: `SessionCreateRequest`
- Validate project access via `resolve_project_access(body.project_id, user, db)`
- Call `AgentService().create_session(db, user.id, body.project_id, body.canvas_id, body.title, body.model_name, body.provider)`
- Return `SessionResponse`

`GET /agent/sessions` — List sessions:
- Query params: `project_id: str`, `canvas_id: str | None = None`
- Validate project access
- Call `AgentService().list_sessions(db, user.id, project_id, canvas_id)`
- Return `SessionListResponse`

`GET /agent/sessions/{session_id}` — Get session detail:
- Validate ownership via `AgentService().get_session(db, session_id, user.id)`
- Return `SessionResponse`

`DELETE /agent/sessions/{session_id}` — Delete session:
- Validate ownership, call `AgentService().delete_session(db, session_id, user.id)`
- Return `{"ok": True}`

`GET /agent/sessions/{session_id}/messages` — Get message history:
- Query params: `limit: int = 50`, `offset: int = 0`
- Query AgentMessage by session_id, ordered by created_at, with limit/offset
- Return `MessageListResponse`

**SSE Chat endpoint per D5:**

`POST /agent/chat/{session_id}` — Streaming agent chat:
- Request body: `ChatRequest`
- Response: `EventSourceResponse` from `sse_starlette.sse`
- Implementation flow:
  1. Load session via `agent_service.get_session()`
  2. Load message history via `agent_service.load_message_history()`
  3. Build `SkillToolset` from `skill_registry` with `SkillContext(user_id=user.id, project_id=session.project_id, canvas_id=session.canvas_id, agent_session_id=session_id, trigger_source="agent")`
  4. Create agent via `agent_service.create_agent(session.provider, session.model_name)`
  5. Build `AgentDeps(user_id=user.id, project_id=session.project_id, canvas_id=session.canvas_id, session_id=session_id, db=db, registry=skill_registry, skill_context=context)`
  6. Define `async def event_generator()`:
     - `yield sse_thinking("analyzing")`
     - `async with agent.iter(user_prompt=body.message, message_history=history, deps=deps, toolsets=[toolset]) as run:`
       - `async for node in run:`
         - If `Agent.is_model_request_node(node)`: yield `sse_thinking("processing")`; then `async with node.stream(run.ctx) as stream:` iterate `PartDeltaEvent` for `TextPartDelta` and yield `sse_token(delta.content_delta)`
         - If `Agent.is_call_tools_node(node)`: for each `ToolCallPart` in `node.model_response.parts`, yield `sse_tool_call(part.tool_name, part.args, part.tool_call_id)`; then for tool results after execution, yield `sse_tool_result(...)`
     - After loop: get `run.result`, yield `sse_done(result.output, usage_dict)`
     - Save messages via `agent_service.save_messages()`
  7. `try/except` wrapper: on any exception yield `sse_error(str(e))` then return
  8. Return `EventSourceResponse(event_generator(), media_type="text/event-stream")`

Import `from sse_starlette.sse import EventSourceResponse` (not manual StreamingResponse).
Import `from pydantic_ai import Agent` for `Agent.is_model_request_node()` / `Agent.is_call_tools_node()`.
Import `from pydantic_ai.messages import ToolCallPart, TextPart` for part type checks.
Import streaming types: `from pydantic_ai.result import PartDeltaEvent, TextPartDelta` (check actual import path from pydantic_ai docs).

**1. Create `api/app/agent/pipeline_tools.py`** — Pipeline Tool per D6 (mixed mode):

Define a PydanticAI-compatible tool function (not a class — PydanticAI tools are async functions that receive `RunContext`):

```python
from pydantic_ai import RunContext
from app.agent.agent_service import AgentDeps

EPISODE_PIPELINE_STEPS = [
    "script.split_clips",
    "script.convert_screenplay",
    "storyboard.plan",
    "storyboard.detail",
]

async def run_episode_pipeline(
    ctx: RunContext[AgentDeps],
    story_text: str,
    steps: list[str] | None = None,
) -> str:
    """Execute a deterministic episode production pipeline: script splitting → screenplay conversion → storyboard planning → storyboard detail.

    Args:
        story_text: The story/script text to process
        steps: Optional list of step names to run (default: all steps). Valid names: script_split_clips, script_convert_screenplay, storyboard_plan, storyboard_detail
    """
```

Implementation:
1. Map requested step names (underscored) back to dotted skill names
2. For each step in order:
   - Build params from previous step results (chain outputs → inputs)
   - `await ctx.deps.registry.invoke(skill_name, params, ctx.deps.skill_context)`
   - Store result in `results[step_name]`
   - If any step fails, return partial results with error info
3. Return JSON summary: `{"completed_steps": [...], "total_steps": N, "results_summary": {...}}`

Step parameter chaining logic:
- `script.split_clips`: params = `{"text": story_text}`
- `script.convert_screenplay`: params = `{"clips": results["script.split_clips"].data.get("clips", [])}`
- `storyboard.plan`: params = `{"screenplay": results["script.convert_screenplay"].data.get("screenplay", "")}`
- `storyboard.detail`: params = `{"shot_plan": results["storyboard.plan"].data.get("shots", [])}`

Create `PIPELINE_TOOL_SCHEMA` dict (JSON Schema for the function parameters) so SkillToolset or manual registration can expose it.

Also create a helper `get_pipeline_toolset()` that returns a `FunctionToolset` wrapping this pipeline tool:
```python
from pydantic_ai.toolsets import FunctionToolset

def get_pipeline_toolset() -> FunctionToolset:
    toolset = FunctionToolset[AgentDeps]()
    toolset.tool(run_episode_pipeline)
    return toolset
```
