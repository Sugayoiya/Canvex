## Screen Specifications

### Screen 1: Enhanced Dashboard (`admin/page.tsx`)

**Layout:** Single column, no scroll (REQ-25: fits one viewport).

**Structure (top to bottom):**

```
┌─ Page Header ─────────────────────────────────────────────────┐
│  h1 "Dashboard" + subtitle + [Refresh] button                │
├─ KPI Grid (2×2) ──────────────────────────────────────────────┤
│  [Total Users]  [Active Teams]                                │
│  [Active Tasks] [Total Cost]                                  │
│  Each card: clickable, alert badge if anomaly detected        │
├─ Activity Overview ───────────────────────────────────────────┤
│  Section heading + WindowToggle (24h|7d|30d)                  │
│  [Tasks] [Failed] [Cost] — 3-column stat card grid           │
├─ AI Providers ────────────────────────────────────────────────┤
│  [Enabled ●] [Disabled ●] — 2-column status cards            │
└───────────────────────────────────────────────────────────────┘
```

**KPI Card Interaction (D-01):**
- `cursor: pointer` on hover
- Hover effect: `border-color` transitions to `var(--ob-glass-hover-border)` (rgba(0,209,255,0.3))
- Hover effect: `transform: translateY(-1px)` with `transition: all 150ms ease`
- Click navigates via `router.push(href)` (Next.js App Router)
- Alert badge positioned in existing badge slot (bottom-left of card, next to description)
- Alert badge uses destructive color when count > 0, neutral otherwise

**KPI Card → Route Mapping:**

| Card | Route | Alert Badge Source |
|------|-------|--------------------|
| Total Users | `/admin/users` | `alerts.quota_warning_users` — "N at quota limit" |
| Active Teams | `/admin/teams` | None (neutral) |
| Active Tasks | `/admin/monitoring` | `alerts.failed_tasks_24h` — "N failed in 24h" |
| Total Cost | `/admin/monitoring?tab=usage` | None (neutral) |

**Data Sources:**
- `GET /admin/dashboard` — KPI values + window stats (existing)
- `GET /admin/alerts` — alert badge counts (new, D-05)
- Parallel requests, independent loading states

## Interaction Specifications

### KPI Card Hover/Click (D-01)

```
Default state:
  border: 1px solid var(--cv4-border-subtle)
  transform: none
  cursor: default → cursor: pointer

Hover state:
  border: 1px solid var(--ob-glass-hover-border)  /* rgba(0,209,255,0.3) */
  transform: translateY(-1px)
  transition: all 150ms ease

Active/pressed state:
  transform: translateY(0)
  transition: all 50ms ease
```

### Tab Switching (D-02)

- TabBar `onTabChange` → `setActiveTab(id)`
- Each tab content lazy-loads its own data via React Query (queries only fire when tab is active)
- No URL persistence — `useState` only
- Tab content unmounts/remounts on switch (no background caching)

### Filter Behavior

- Search input: 300ms debounce before API call (consuming page handles debounce, per Phase 09 D-09)
- Status/provider dropdowns: immediate filter on change
- All filters reset when switching tabs
- Pagination resets to page 1 on any filter change

### Data Refresh

- Dashboard: `refetchInterval: 60_000` (existing, keep)
- Monitoring tabs: no auto-refresh (manual via React Query refetch)
- Dashboard alerts: `refetchInterval: 60_000` (match dashboard refresh)

## Backend Contract: GET /admin/alerts

**Endpoint:** `GET /admin/alerts`
**Auth:** `is_admin` required
**Response:**

```json
{
  "quota_warning_users": 3,
  "failed_tasks_24h": 5,
  "error_providers": 1
}
```

| Field | Type | Description |
|-------|------|-------------|
| `quota_warning_users` | int | Users where any quota dimension ≥ 80% used |
| `failed_tasks_24h` | int | Tasks with status=failed in last 24 hours |
| `error_providers` | int | System-scope providers with `is_enabled=false` or last call errored |

**Thresholds:**
- `quota_warning_users > 0` → show destructive badge on Total Users card
- `failed_tasks_24h > 0` → show destructive badge on Active Tasks card
- `error_providers` — not surfaced on dashboard cards (provider status section already shows this)
