**Per D-01 and D-05, make these changes to `web/src/app/admin/page.tsx`:**

**1) Add imports:**
```typescript
import { useRouter } from "next/navigation";
```

**2) Add alerts interface and query:**
```typescript
interface AlertsData {
  quota_warning_users: number;
  failed_tasks_24h: number;
  error_providers: number;
}
```

In `AdminDashboardPage`, add a second parallel query after the existing dashboard query:
```typescript
const router = useRouter();

const alertsQuery = useQuery({
  queryKey: ["admin", "alerts"],
  queryFn: () => adminApi.getAlerts().then((r) => r.data as AlertsData),
  refetchInterval: 60_000,
});
const alerts = alertsQuery.data;
```

**3) Extend KpiCard props with `href` and `alertBadge`:**

Add to the KpiCard destructured props:
```typescript
href?: string;
alertBadge?: string | null;
alertBadgeVariant?: "destructive" | "neutral";
```

Add these style/behavior changes to the KpiCard root `<div>`:
- `cursor: href ? "pointer" : "default"`
- `transition: "all 150ms ease"`
- `onClick={() => href && router.push(href)}`
- `role={href ? "link" : undefined}`
- `tabIndex={href ? 0 : undefined}`
- `onKeyDown={(e) => { if (href && e.key === "Enter") router.push(href); }}`
- `onMouseEnter`: if `href`, set `borderColor` to `"rgba(0,209,255,0.3)"` and `transform` to `"translateY(-1px)"`
- `onMouseLeave`: reset `borderColor` to `"var(--cv4-border-subtle)"` and `transform` to `"none"`

**4) Add alert badge rendering inside KpiCard:**

After the existing `badge` span, add conditional alert badge:
```typescript
{alertBadge && (
  <span style={{
    fontFamily: "var(--font-headline)", fontSize: 12, fontWeight: 400,
    lineHeight: 1.3, padding: "2px 8px", borderRadius: 6,
    background: "#FFB4AB15", border: "1px solid #FFB4AB30", color: "#FFB4AB",
  }}>
    {alertBadge}
  </span>
)}
```

**5) Wire KPI cards with href and alertBadge per mapping table:**

- Total Users card: `href="/admin/users"`, `alertBadge={alerts && alerts.quota_warning_users > 0 ? \`${alerts.quota_warning_users} at quota limit\` : null}`
- Active Teams card: `href="/admin/teams"`, no alertBadge
- Active Tasks card: `href="/admin/monitoring"`, `alertBadge={alerts && alerts.failed_tasks_24h > 0 ? \`${alerts.failed_tasks_24h} failed in 24h\` : null}`
- Total Cost card: `href="/admin/monitoring"`, no alertBadge (per D-04, no charts on dashboard)

**6) Pass `router` context:** `router` is declared at top of `AdminDashboardPage`, passed implicitly via closure since `KpiCard` is a child component in the same file. No prop drilling needed — move `KpiCard` to accept `onClick` or use `router` from the parent's scope. Simplest: add `onClick?: () => void` prop to KpiCard and call `router.push(href)` at the parent level.

Actually, since KpiCard is defined in the same file, the cleanest approach: add `href` prop, have KpiCard call the router internally using a hook. But KpiCard is not a top-level component with hooks access. Best approach: pass `onClick` from parent.

Final approach for KpiCard:
- Add `onClick?: () => void` prop
- Root div: `onClick={onClick}`, `cursor: onClick ? "pointer" : "default"`, hover effects when `onClick` is defined
- In the KPI grid, wrap each card: `onClick={() => router.push("/admin/users")}` etc.
