### Focal Point

On initial canvas load, the user's eye is drawn to (in priority order):

1. **Center viewport node** — The most recently created or first node, positioned near viewport center. Node card surface (`surface.primary`) contrasts against the dominant canvas background (`canvas.background`), creating an immediate figure-ground separation.
2. **Left floating menu "+" button** — The only menu item with a filled background (`surface.primary`) and border, visually heavier than other icon-only menu buttons. Guides new users to the primary creation action.
3. **AI Chat popup header** (if open) — "Cinematic AI" title at `chatTitle` weight (700) is the boldest text element visible, anchoring the Agent interaction area.

Destructive color: Not defined in current tokens. Use `#EF4444` (dark) / `#DC2626` (light) for delete actions — node deletion, asset deletion. Applied to icon + text only, never as background fill.

## Interaction Contracts

### Focus Panel System (Core Interaction)

| Node State | Node Type | Panel | Direction | Gap |
|------------|-----------|-------|-----------|-----|
| Empty (no content) | All types | AIGeneratePanel | Below node ↓ | 12px |
| Has content | text | TextToolbar | Above node ↑ | 8px |
| Has content | image | TemplateMenu | Above node ↑ | 12px |
| Has content | video | TemplateMenu | Above node ↑ | 12px |
| Has content | audio | TemplateMenu | Above node ↑ | 12px |

**Panel positioning algorithm:**
```
hasContent → panel at (node.x, node.y - panelHeight - gap)
empty      → panel at (node.x, node.y + nodeHeight + gap)
```

**Dismiss:** Click outside node + panel area, or press Escape.

### Node Connection Rules

| Source | Valid Targets |
|--------|--------------|
| text | text, image, video, audio |
| image | image, video |
| video | video |
| audio | audio, video |

**Edge drop behavior:** Dragging from handle → release on empty canvas → show NodeCreationMenu with compatible types only.

### Left Floating Menu Interactions

| Button | Icon | Size | Action |
|--------|------|------|--------|
| Add node | `plus` (18px) | 36×36px | Show node type picker (text/image/video/audio), create at viewport center |
| Workflow | `git-branch` (16px) | 36×36px | Toggle workflow/branch view |
| Assets | `package` (16px) | 36×36px | Toggle AssetPanel sidebar |
| History | `history` (16px) | 36×36px | Toggle version history |
| Help | `info` (16px) | 36×36px | Toggle help panel |
| User | Letter avatar (12px) | 36×36px | User menu |

### Template Application Flow

1. User focuses a node with content → TemplateMenu appears above
2. User clicks template chip (e.g. "九宫格场景")
3. System creates downstream node at `(source.x + offset_x, source.y + offset_y)`
4. New node inherits hidden_prompt + auto-connects edge from source
5. New node appears in empty state with AIGeneratePanel → user clicks generate

### AI Generate Panel Parameters (per node type)

| Node Type | Parameters Shown |
|-----------|-----------------|
| text | Model selector (Provider → Model) |
| image | Model selector, aspect ratio + resolution, generation count |
| video | Mode selector (text-to-video / image-to-video), aspect ratio, duration, camera control |
| audio | (reserved for future) |

### Quota Indicator

Token/credit consumption shown in AI Generate Panel bottom-bar:
- `⚡` zap icon (12px, `text.disabled`) + count ("14", 12px, `text.disabled`)
- Position: right side of bottom-bar, before send button, separated by 1×16px divider

When quota exceeded:
- Send button disabled (50% opacity)
- Tooltip: "额度已用完"
- Zap icon turns `#EF4444`

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Keyboard navigation | Tab through left menu items, Enter to activate, Escape to dismiss panels |
| Focus trap | AI Generate Panel and AI Chat Popup trap focus when open |
| Screen reader | `aria-label` on all icon-only buttons, `role="toolbar"` on TextToolbar, `role="dialog"` on Chat Popup |
| Reduced motion | Respect `prefers-reduced-motion: reduce` — disable all animations, transitions snap instantly |
| Color contrast | All `text.primary` on `surface.primary` passes WCAG AA (dark: #E5E2E1 on #2A2A2A = 9.3:1, light: #1A1A1A on #FFFFFF = 17.4:1) |
| Touch targets | All interactive buttons minimum 36×36px (left menu), send button 28-30px with adequate spacing |
