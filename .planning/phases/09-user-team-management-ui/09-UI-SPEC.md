## Spacing Scale (inherited — no changes)

Inherited from Phase 08 UI-SPEC: {4, 8, 16, 24, 32, 48, 64} — all multiples of 4.

### Phase 09 Fixed Dimensions

| Element | Value | Rationale |
|---------|-------|-----------|
| Table row height | 48px | Standard set value, comfortable touch target |
| Table header height | 40px | Distinguishes header from data rows |
| Search input height | 36px | Matches Phase 08 button height |
| Filter dropdown height | 36px | Matches search input |
| Dropdown menu item height | 36px | Matches nav item height from Phase 08 |
| Dropdown menu width | 200px | Fits longest label "Revoke Admin Role" |
| Confirmation modal width | 420px | Fits warning text + 2-button footer comfortably |
| Badge height | 24px | Compact, readable at 12px font |
| Badge padding | 4px 8px | Horizontal padding for label text |
| Toolbar gap (search/filters) | 16px | md spacing between search and filter controls |
| Toolbar-to-table gap | 16px | md spacing between filter bar and table |
| Table cell horizontal padding | 16px | md padding for readable cell content |
| Table cell vertical padding | 12px | Card-padding-derived (nearest standard: 3×4=12) |
| Pagination bar height | 48px | Matches table row cadence |
| Pagination button size | 32px × 32px | Touch-friendly, fits 2-digit page numbers |

Exceptions: `12px` cell vertical padding (3×4) — not in the standard set but required for table density. This is the only non-standard value.

## Accessibility

| Element | Requirement |
|---------|-------------|
| Data table | `<table>` with `role="table"`, `<thead>`/`<tbody>` semantics |
| Table header cells | `<th scope="col">` |
| Sortable headers | `aria-sort="ascending"` / `"descending"` / `"none"`, `role="columnheader"`, clickable via button |
| Status badge | `aria-label="Status: Active"` (screen reader context) |
| Admin badge | `aria-label="Role: Admin"` |
| Dropdown trigger | `aria-haspopup="menu"`, `aria-expanded="true/false"`, `aria-label="User actions for {email}"` |
| Dropdown menu | `role="menu"`, items have `role="menuitem"` |
| Disabled menu item | `aria-disabled="true"` + tooltip text in `aria-label` |
| Modal | `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to title |
| Modal focus trap | Focus trapped within modal while open; return focus to trigger on close |
| Search input | `role="searchbox"`, `aria-label="Search users"` |

### React Query Keys

| Query | Key | Params |
|-------|-----|--------|
| User list | `['admin', 'users', { q, status, is_admin, sort_by, sort_order, limit, offset }]` | All filter/sort/pagination state |
| Team list | `['admin', 'teams', { q, limit, offset }]` | Search + pagination state |

### Mutations

| Mutation | Endpoint | Invalidates |
|----------|----------|-------------|
| Toggle status | `adminApi.toggleUserStatus(id, { status })` | `['admin', 'users']` |
| Toggle admin | `adminApi.toggleUserAdmin(id, { is_admin })` | `['admin', 'users']` |

### State Management

- **Server state:** React Query (all table data, pagination, filters)
- **Client state:** React `useState` for:
  - Search input value (controlled, debounced before query)
  - Filter selections
  - Currently open dropdown menu (which row)
  - Modal open/close + modal action context (which user, which action)
  - Current page number
  - Current sort column + direction
- **No Zustand** needed for Phase 09 — all state is page-local

### URL State Sync

Not required for Phase 09. All filter/sort/pagination state is component-local. URL state sync (e.g., `?status=banned&page=2`) deferred as enhancement.

## Backend Extension Required (D-08)

`AdminUserListItem` schema currently lacks `teams` field. Phase 09 requires:

1. **Backend:** Add `teams` field to `AdminUserListItem` in `api/app/schemas/admin.py` — list of team names (or count)
2. **Backend:** Update `GET /admin/users` query in `api/app/api/v1/admin_users.py` to JOIN team_members → teams
3. **Frontend:** Update `adminApi.listUsers` TypeScript types to include `teams` field

Recommended schema addition:
```
teams: list[str]  # team names the user belongs to
```

Display in table: If 0 teams → "—" (`--cv4-text-muted`). If 1–2 teams → comma-separated names. If 3+ teams → "{first_name}, +{N} more" with tooltip showing full list.
