## Interaction Contract

### Data Fetching

| Data | Source | Cache Key | Refetch Strategy |
|------|--------|-----------|------------------|
| Provider list + keys | `GET /api/v1/ai-providers?owner_type=system` | `["admin", "providers", { owner_type: "system" }]` | Existing вҖ” invalidate on mutations |
| Key health (per key) | `GET /api/v1/ai-providers/{id}/keys/{key_id}/health` | `["admin", "key-health", key_id]` | Fetch on card expand, poll every 30s while expanded |
| Key usage trend | Included in health response `.usage_trend` | Same as above | Same as above |
| Key error history | Included in health response `.recent_errors` | Same as above | Same as above |

### State Management

- Health data fetched lazily: only when provider card is expanded AND key detail row is expanded
- Use React Query `useQuery` with `enabled: isExpanded` pattern
- Polling interval: 30,000ms while detail is visible, stopped when collapsed
- Optimistic update on toggle/reset: immediately update local state, revert on error

### Loading States

| State | Visual |
|-------|--------|
| Health loading | HealthBadge shows pulsing gray placeholder (same animation as provider card skeleton) |
| Sparkline loading | Gray rectangular placeholder 192Г—40px with pulse animation |
| Error list loading | Two gray line placeholders with pulse animation |
| Toggle pending | Toggle switch at 70% opacity, cursor not-allowed |
| Reset pending | "Reset Errors" text replaced with "..." |

### Error States

| Error | Visual | Recovery |
|-------|--------|----------|
| Health fetch failed | HealthBadge shows "вҖ”" in gray, no detail panel | Auto-retry on next poll cycle |
| Toggle failed | Toast: "еҲҮжҚў Key зҠ¶жҖҒеӨұиҙҘ: {error}" | Revert optimistic state |
| Reset failed | Toast: "йҮҚзҪ®й”ҷиҜҜи®Ўж•°еӨұиҙҘ: {error}" | No state change |

## API Contract (Frontend Expectations)

### `GET /api/v1/ai-providers/{config_id}/keys/{key_id}/health`

```typescript
interface KeyHealthResponse {
  key_id: string;
  error_count: number;
  last_used_at: string | null;       // ISO 8601
  is_healthy: boolean;
  health_badge: "healthy" | "degraded" | "unhealthy";
  recent_errors: Array<{
    type: string;                     // e.g. "RateLimitError", "InternalServerError"
    message: string;                  // truncated to 200 chars
    at: string;                       // ISO 8601
  }>;
  usage_trend: Array<{
    hour: string;                     // "YYYYMMDDHH" format
    count: number;
  }>;                                 // 24 entries, sorted chronologically
}
```

### `PATCH /api/v1/ai-providers/{config_id}/keys/{key_id}`

```typescript
interface KeyUpdateRequest {
  is_active?: boolean;
  reset_error_count?: boolean;
}
```
