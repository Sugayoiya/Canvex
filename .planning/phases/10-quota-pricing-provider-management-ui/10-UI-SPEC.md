## Spacing Scale (inherited — no changes)

Inherited from Phase 08 UI-SPEC: {4, 8, 16, 24, 32, 48, 64} — all multiples of 4.

### Phase 10 Fixed Dimensions

| Element | Value | Rationale |
|---------|-------|-----------|
| Tab bar height | 40px | Consistent with table header from Phase 09 |
| Tab item padding | 0 16px | Standard md horizontal padding |
| Expandable row detail area padding | 24px | Card-level padding (Phase 08) |
| Progress bar height | 8px | Compact, 2×4=8, standard set |
| Progress bar border-radius | 4px | xs radius for pill shape |
| Quota input height | 36px | Matches Phase 09 search/button height |
| Quota input width | 120px | Fits 6-digit numbers comfortably |
| Provider card min-height | 80px | Collapsed card: icon + name + status + actions |
| Provider card expanded padding | 24px | Card-level padding (Phase 08) |
| Key table row height | 44px | Slightly less than 48px for denser key list |
| Key input area gap | 16px | md spacing between label/key inputs |
| Pricing modal width | 520px | Wider than ConfirmationModal (420px) to fit form fields |
| Pricing modal max-height | 80vh | Scrollable if content exceeds viewport |
| Summary card height | 80px | Compact info card at top of Pricing page |
| Summary card gap | 16px | md spacing between cards |
| Org total bar height | 48px | Matches pagination bar cadence |

Exceptions: `44px` key table row (11×4) — compact variant for nested key list. `80px` summary card (20×4) — compact info card. `12px` cell vertical padding (3×4, inherited from Phase 09). All multiples of 4.

## Data Flow Contract

### React Query Keys

| Query | Key | Params |
|-------|-----|--------|
| User list (for quota) | `['admin', 'users', { q, limit, offset }]` | Search + pagination |
| Team list (for quota) | `['admin', 'teams', { q, limit, offset }]` | Search + pagination |
| User quota | `['admin', 'quota', 'user', userId]` | Per-user quota |
| Team quota | `['admin', 'quota', 'team', teamId]` | Per-team quota |
| Pricing list | `['admin', 'pricing', { active_only }]` | Status filter |
| Provider list (system) | `['admin', 'providers', { owner_type: 'system' }]` | System-scope only |
| Provider keys | `['admin', 'providers', providerId, 'keys']` | Per-provider keys |

### Mutations

| Mutation | Endpoint | Invalidates |
|----------|----------|-------------|
| Update user quota | `quotaApi.updateUserQuota(id, data)` | `['admin', 'quota', 'user', id]` |
| Update team quota | `quotaApi.updateTeamQuota(id, data)` | `['admin', 'quota', 'team', id]` |
| Create pricing | `billingApi.createPricing(data)` | `['admin', 'pricing']` |
| Update pricing | `billingApi.updatePricing(id, data)` | `['admin', 'pricing']` |
| Delete pricing | `billingApi.deletePricing(id)` | `['admin', 'pricing']` |
| Create provider | `aiProvidersApi.create(data)` | `['admin', 'providers']` |
| Update provider | `aiProvidersApi.update(id, data)` | `['admin', 'providers']` |
| Delete provider | `aiProvidersApi.delete(id)` | `['admin', 'providers']` |
| Add key | `aiProvidersApi.addKey(id, data)` | `['admin', 'providers', id, 'keys']`, `['admin', 'providers']` |
| Delete key | `aiProvidersApi.deleteKey(id, keyId)` | `['admin', 'providers', id, 'keys']`, `['admin', 'providers']` |

### API Client Extension Required (D-20)

Add to `billingApi` in `web/src/lib/api.ts`:

```
billingApi.createPricing(data)   → POST /billing/pricing/
billingApi.updatePricing(id, data) → PATCH /billing/pricing/{id}
billingApi.deletePricing(id)     → DELETE /billing/pricing/{id}
```

### State Management

- **Server state:** React Query (all list data, individual quota data, pricing list, provider list)
- **Client state:** React `useState` for:
  - Active tab (users/teams) on Quotas page
  - Search input value (controlled, 300ms debounce)
  - Expanded row ID (one at a time, per page)
  - Quota edit form values (per expanded row)
  - Pricing status filter
  - PricingFormModal open/close + mode (create/edit) + initial data
  - ProviderFormModal open/close + mode (create/edit) + initial data
  - ConfirmationModal open/close + action context
  - Add key form values (label + api_key)
  - Current page number (per tab on quotas, per page on pricing)
- **No Zustand** needed for Phase 10 — all state is page-local

### URL State Sync

Not required for Phase 10. Tab, filter, and pagination state is component-local.
