**All 5 canvas nodes wired with useUpstreamData for edge-based input, useNodePersistence for config auto-save, and coordinated execution writeback with cancelPending race prevention**

## Accomplishments

- Wired `useUpstreamData` into LLM, ImageGen, Extract, and Output nodes so they read typed input from connected edges
- Wired `useNodePersistence` into LLM, ImageGen, Extract, and TextInput nodes for debounced config auto-save
- Added `onComplete` callback parameter to `useNodeExecution` hook for ReactFlow data sync after execution
- Added `canvasApi.updateNode` writeback to `useNodeExecution` for completed/failed/blocked/timeout statuses with graceful `.catch` degradation
- Replaced Extract node's inline `useEdges`/`useNodes`/`useMemo` with shared `useUpstreamData` hook
- TextInput node now persists text with 2s debounce + immediate flush on blur
- Output node auto-aggregates upstream text/imageUrl/json arrays with count badges and backward-compatible fallback
- Applied correct hook ordering (checker warning) to prevent TDZ: `useReactFlow → useUpstreamData → useNodePersistence → handleExecutionComplete → useNodeExecution`

## Decisions Made

- **Hook order for TDZ safety:** `useReactFlow → useUpstreamData → useNodePersistence → handleExecutionComplete → useNodeExecution` — the checker warning identified that the plan's original order would cause a TDZ because `handleExecutionComplete` references `persistence.cancelPending()` which must be declared first
- **Writeback graceful degradation:** `canvasApi.updateNode(...).catch(console.warn)` — execution state lives locally in React first, backend sync is best-effort and non-blocking
- **Output backward compatibility:** When `upstream.*` arrays are empty, Output falls back to direct `nodeData.*` fields so persisted execution results still display

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed hook ordering to prevent TDZ (checker warning)**
- **Found during:** Task 1
- **Issue:** Plan suggested defining `handleExecutionComplete` before `useNodePersistence`, but the callback references `persistence.cancelPending()`. This would cause a TDZ ReferenceError because `const persistence` from `useNodePersistence()` wouldn't be declared yet when `useCallback` evaluates its dependency array.
- **Fix:** Reordered hooks in all three execution nodes: `useReactFlow → useUpstreamData → useNodePersistence → handleExecutionComplete → useNodeExecution`
- **Files modified:** `llm-node.tsx`, `image-gen-node.tsx`, `extract-node.tsx`
- **Committed in:** `f4d15d1`

**2. [Rule 1 - Bug] Added timeout/blocked writeback to useNodeExecution**
- **Found during:** Task 1
- **Issue:** Plan only showed writeback for completed/failed statuses, but timeout and blocked statuses also need backend persistence
- **Fix:** Added `canvasApi.updateNode` writeback for timeout (in poll) and blocked (in both poll and execute) statuses
- **Files modified:** `web/src/components/canvas/hooks/use-node-execution.ts`
- **Committed in:** `f4d15d1`

**3. [Rule 1 - Bug] Fixed TypeScript type error in Output node jsons array**
- **Found during:** Task 2
- **Issue:** `jsons` array inferred as `(object | unknown)[]` due to `nodeData.json` being `unknown`, causing TS2322 in JSX rendering
- **Fix:** Added explicit `object[]` type annotation with `as object[]` cast on fallback
- **Files modified:** `web/src/components/canvas/nodes/output-node.tsx`
- **Committed in:** `eba3682`
