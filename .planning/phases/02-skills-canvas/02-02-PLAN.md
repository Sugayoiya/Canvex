**api/app/api/v1/canvas.py:**

Create APIRouter with prefix="/canvas", tags=["canvas"]. CRITICAL: Every endpoint that accesses a canvas MUST call `resolve_project_access()` to verify the user has at least "editor" role on the canvas's project. This prevents IDOR attacks.

**Authorization pattern for canvas endpoints:**
1. For create: Accept `project_id` in body → call `await resolve_project_access(req.project_id, user, db)` before creating canvas.
2. For list: Accept `project_id` query param → call `await resolve_project_access(project_id, user, db)` then filter canvases by that project.
3. For get/update/delete canvas by id: Load canvas first → call `await resolve_project_access(canvas.project_id, user, db)`.
4. For node/edge operations: Load parent canvas → call `await resolve_project_access(canvas.project_id, user, db)`.

Helper function `_get_canvas_with_access(canvas_id, user, db)`:
```python
async def _get_canvas_with_access(canvas_id: str, user, db: AsyncSession) -> Canvas:
    result = await db.execute(select(Canvas).where(Canvas.id == canvas_id, Canvas.is_deleted == False))
    canvas = result.scalar_one_or_none()
    if not canvas:
        raise HTTPException(status_code=404, detail="Canvas not found")
    await resolve_project_access(canvas.project_id, user, db)
    return canvas
```

10 endpoints — same CRUD as before:
1. `POST /` — Create canvas (resolve project access first)
2. `GET /` — List canvases for project (resolve project access on project_id param)
3. `GET /{canvas_id}` — Get canvas with nodes and edges (resolve via helper)
4. `PATCH /{canvas_id}` — Update canvas metadata

6. `POST /{canvas_id}/nodes` — Add node

9. `POST /{canvas_id}/edges` — Add edge (validate both nodes exist in canvas)
10. `DELETE /{canvas_id}/edges/{edge_id}` — Delete edge
