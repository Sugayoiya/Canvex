**api/app/api/v1/billing.py:**

APIRouter with prefix="/billing", tags=["billing"]. Addresses review concern #1: admin mutations have EXPLICIT `require_admin(user)` calls.

1. `GET /pricing` — List active pricing. Auth: `get_current_user` (any authenticated user). Optional filters: `provider`, `model_type`.
2. `GET /pricing/{pricing_id}` — Get single pricing. Auth: `get_current_user`.
3. `POST /pricing` — Create pricing. Auth: `get_current_user` THEN `require_admin(user)`. Body: `ModelPricingCreate`. Return 201.
4. `PATCH /pricing/{pricing_id}` — Update pricing. Auth: `get_current_user` THEN `require_admin(user)`. Body: `ModelPricingUpdate`. Use `model_dump(exclude_unset=True)` + `setattr`.
5. `DELETE /pricing/{pricing_id}` — Deactivate pricing. Auth: `get_current_user` THEN `require_admin(user)`. Set `is_active = False` (soft-deactivate, not hard delete). Return `{"ok": True}`.
6. `GET /usage/stats` — User's AI call stats. Auth: `get_current_user`. Aggregate from AICallLog: `count(*)`, `sum(input_tokens)`, `sum(output_tokens)`, `sum(cost)` WHERE `user_id = user.id`. Return dict.
