<domain>
## Phase Boundary

将 Canvex AI 架构从 PydanticAI + Skill-Centric 模式重构为 LangChain + Anthropic Agent Skills 模式：

- Agent 成为唯一 LLM 推理者（使用 LangChain `create_react_agent`）
- Skill 变为声明式 SKILL.md 目录（Anthropic 三级渐进加载）
- Tools 变为 `@tool` 纯函数（数据读写 + 外部 AI 调用）
- 支持多供应商 LLM 切换（Gemini / OpenAI / DeepSeek）
- 集成 LangSmith 全链路追踪
- 前端所有 AI 操作统一走 `POST /agent/chat` SSE
- 清除 PydanticAI 及旧 Skill handler 代码

**不包含在此 Phase 范围内：**
- Phase 13-16 的 SkillDescriptor / ArtifactStore / QueryEngine / Admin Skills（将在 12.1 完成后重新评估）
- 业务 Skill 扩展（v3.1 范围，SKIL-01~04）
- SubAgentTool / 跨会话记忆等高级 Agent 能力（v3.1 范围）

</domain>

<decisions>
## Implementation Decisions

### 框架选型：LangChain 替代 PydanticAI
- **LangChain** 统一 `BaseChatModel` 接口，多供应商切换仅需更换一个构造调用
- 使用 `create_react_agent`（来自 `langgraph.prebuilt`）替代 PydanticAI 自定义 agent loop
- `astream_events` 原生支持 SSE 流式
- LangSmith 通过环境变量自动启用追踪

### 依赖变更
- **移除**: `pydantic-ai`, `pydantic-ai-slim`
- **新增**: `langchain-core>=0.3`, `langchain-google-genai>=2.0`, `langchain-openai>=0.3`, `langgraph>=0.3`, `langsmith>=0.2`

### Tool 分层架构
- **读工具** (`context_tools.py` 改造): `get_script`, `get_characters`, `get_scenes`, `get_shots`, `get_project_info`, `get_episodes`, `get_style_templates`
- **写工具** (新增 `mutation_tools.py`): `save_characters`, `save_scenes`, `save_screenplay`, `save_shot_plan`, `save_shot_details`, `update_shot`
- **外部 AI 工具** (新增 `ai_tools.py`): `generate_image`, `generate_video` — 包装 ProviderManager
- **Skill 工具** (新增 `skill_tools.py`): `read_skill(name)` (Level 2), `read_resource(name, file)` (Level 3)
- 所有工具使用 LangChain `@tool` 装饰器 + Pydantic `BaseModel` 作为 `args_schema`

### ProviderManager 扩展
- 新增 `resolve_langchain_llm(provider_name, model_name, team_id)` → 返回 `BaseChatModel`
- 复用现有 `resolve_api_key()` 的 3 级回退链（team → global → env）
- Agent 拿到 `BaseChatModel` 实例，不接触裸 API key
- 映射表: gemini → `ChatGoogleGenerativeAI`, openai → `ChatOpenAI`, deepseek → `ChatOpenAI` (custom base_url)

### Anthropic SKILL.md 目录规范
- 目录结构: `api/app/agent/skills/{skill-name}/SKILL.md` + 可选 `REFERENCE.md` + `scripts/`
- YAML frontmatter: `name` (小写+连字符, ≤64 chars) + `description` (≤1024 chars, WHAT + WHEN)
- 10 个 SKILL.md 目录: extract-characters, extract-scenes, split-clips, convert-screenplay, create-storyboard, detail-storyboard, generate-shot-image, generate-shot-video, episode-pipeline, refine-text

### SkillLoader 三级加载
- Level 1 (元数据): 启动时解析所有 SKILL.md YAML frontmatter → 注入 system prompt
- Level 2 (指令): Agent 调用 `read_skill(name)` 时读取 SKILL.md body
- Level 3 (资源): Agent 调用 `read_resource(name, file)` 按需读取附属文件

### AgentService 重写
- `agent_service.py` 使用 `create_react_agent(llm, tools)` 构建 agent
- System prompt = 基础人设 + SkillLoader L1 元数据片段
- `chat()` 方法通过 `agent.astream_events()` 返回 SSE 流
- 单一入口点: `POST /agent/chat`

### LangSmith 集成
- 仅通过环境变量配置: `LANGCHAIN_TRACING_V2=true`, `LANGCHAIN_API_KEY`, `LANGCHAIN_PROJECT=canvex`
- 零代码侵入，LangChain 自动上报所有推理链

### 前端适配
- 废弃 `POST /skills/invoke` 端点
- 所有 AI 操作改为 `POST /agent/chat` SSE 流式
- `web/src/lib/api.ts`: 删除 `skillsApi.invoke/poll`，新增技能列表接口
- 新增供应商/模型选择 UI（下拉切换 Gemini/OpenAI/DeepSeek）

<specifics>
## Specific Ideas

### Tool 实现示例
```python
from langchain_core.tools import tool
from pydantic import BaseModel, Field

class GetScriptInput(BaseModel):
    episode_id: str = Field(description="Episode ID to fetch script for")

@tool(args_schema=GetScriptInput)
async def get_script(episode_id: str) -> str:
    """Retrieve the screenplay/script content for a specific episode."""
    async with get_db_session() as db:
        script = await db.execute(select(Script).where(Script.episode_id == episode_id))
        return script.scalar_one_or_none()?.content or "No script found"
```

### resolve_langchain_llm 实现模式
```python
async def resolve_langchain_llm(self, provider_name, model_name, team_id=None, **kwargs):
    api_key, owner, key_id = await self.resolve_api_key(provider_name, team_id=team_id)
    if provider_name == "gemini":
        from langchain_google_genai import ChatGoogleGenerativeAI
        return ChatGoogleGenerativeAI(model=model_name, google_api_key=api_key, **kwargs)
    elif provider_name in ("openai", "deepseek"):
        from langchain_openai import ChatOpenAI
        extra = {"base_url": "https://api.deepseek.com/v1"} if provider_name == "deepseek" else {}
        return ChatOpenAI(model=model_name, api_key=api_key, **extra, **kwargs)
```

### AgentService 核心模式
```python
agent = create_react_agent(llm, self.tools)
async for event in agent.astream_events(
    {"messages": [SystemMessage(content=system_prompt), HumanMessage(content=message)]},
    version="v2",
):
    yield event  # SSE 流式
```
