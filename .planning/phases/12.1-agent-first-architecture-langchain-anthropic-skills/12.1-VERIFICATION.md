### 1. Live Agent Chat Streaming

**Test:** Start backend (`cd api && uv run uvicorn app.main:app --reload --port 8000`). Create an agent session via POST `/agent/sessions`. Send a message via POST `/agent/chat/{session_id}` and observe SSE events.
**Expected:** SSE events flow: `thinking` → `token` (streaming text) → optional `tool_call`/`tool_result` → `done`. LangChain agent uses resolved LLM provider.
**Why human:** Requires running backend with live Gemini/OpenAI API credentials.

### 2. Canvas Node Execution (Legacy Path)

**Test:** Open the web app, navigate to a project canvas, trigger an image generation node.
**Expected:** Canvas node executes via legacy `skillsApi.invoke` path (AGENT_CHAT_FOR_CANVAS=false). Image generates successfully.
**Why human:** Requires running full stack (frontend + backend + configured AI provider).

### 3. LangSmith Tracing

**Test:** Set `LANGCHAIN_TRACING_V2=true` and `LANGSMITH_API_KEY=<key>` in api/.env, restart backend, send agent chat message, check LangSmith dashboard.
**Expected:** Full agent trace visible — LLM calls, tool invocations, reasoning chain captured.
**Why human:** Requires LangSmith account and API key to verify trace capture.

### 4. Multi-Provider LLM Switching

**Test:** Create agent sessions with different providers (gemini, openai, deepseek). Send messages via each.
**Expected:** Each provider resolves correctly. Gemini → ChatGoogleGenerativeAI, OpenAI → ChatOpenAI, DeepSeek → ChatOpenAI(base_url=deepseek).
**Why human:** Requires API keys for multiple LLM providers.

### Gaps Summary

**Primary finding:** Phase 12.1 achieves its core goals. LangChain replaces PydanticAI throughout the backend. The 17-tool system, 10-skill SKILL.md library, multi-provider agent engine, and SSE streaming are all in place and properly wired.

**Two known partial items:**

1. **AGENT-04 canvas migration (intentional scope limit):** The plan explicitly acknowledges that canvas node execution stays on the legacy path behind a feature flag. This is a design decision, not a gap — full canvas migration is deferred to Phase 12.1B/13+. The chat-based AI interaction path (the primary user-facing flow) is fully migrated.

2. **LangSmith tracing configuration (minor doc gap):** The `langsmith` dependency is installed and tracing auto-activates via standard LangChain env vars. However, no `.env.example` or documentation guides the user on setting `LANGCHAIN_TRACING_V2=true` / `LANGSMITH_API_KEY` / `LANGCHAIN_PROJECT`. This is a documentation gap, not a code gap.

**Bookkeeping:** REQUIREMENTS.md needs updating to include AGENT-01 through AGENT-05 and remove the "LangChain migration is out of scope" contradiction. This doesn't affect code correctness.
