# Phase 12.2 — UI Design Contract

> Visual and interaction contract for the Provider & Model Preset Management phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian Lens custom properties) |
| Preset | not applicable |
| Component library | none (inline-style React components) |
| Icon library | Lucide React (existing) + new `ProviderIcon` component |
| Font | Space Grotesk (headings, code-like) + Manrope (body, labels) |

**Source:** Existing `globals.css` `--cv4-*` / `--ob-*` tokens. No new CSS custom properties introduced in this phase.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Subtitle margin-top, icon-text micro-gaps |
| sm | 8px | Table cell padding, compact element spacing |
| md-sm | 12px | Input horizontal padding, inter-element gaps |
| md | 16px | Card content padding, card margin-bottom, section gaps |
| lg | 24px | Model list section padding-top, section breaks |
| xl | 32px | Group header vertical margins |
| 3xl | 64px | Empty state vertical padding |

Exceptions: 6px for form label `margin-bottom` and button icon-text `gap` (established pattern in existing `ProviderFormModal` and `PageHeader`).

## Component Inventory

### New Components

| Component | File | Responsibility |
|-----------|------|----------------|
| `ProviderIcon` | `web/src/components/admin/provider-icon.tsx` | Maps provider_name string → inline SVG (gemini / openai / deepseek). Falls back to Lucide `Bot` icon for unknown providers. Size prop: 20px default. |
| `ProviderModelList` | `web/src/components/admin/provider-model-list.tsx` | Collapsible model table rendered within expanded `ProviderCard`, below key management section. Receives `providerId`, fetches models via React Query. Contains model rows + add-model form. |

| Component | File | Changes |
|-----------|------|---------|

| `ProviderCard` | `web/src/components/admin/provider-card.tsx` | Replace `Bot` icon with `ProviderIcon`. Remove `onDelete` prop + delete button from header. Add `ProviderModelList` section below key table in expanded area. Add provider `description` line below display_name in header. |
| `ProviderFormModal` | `web/src/components/admin/provider-form-modal.tsx` | Remove `provider_name`, `display_name`, `priority` fields. Show only: API Key input + optional Base URL input (conditionally shown when `sdk_type === "openai_compatible"`). Title changes to "配置 {provider.display_name}". |

| Component | Usage |
|-----------|-------|

| `AdminErrorBoundary` | Wraps provider list (existing) |
| `UsageSparkline` | Key usage trend in expanded row (existing) |

## Interaction Contracts

### I-01: Page Load — Group Separation

| Property | Value |
|----------|-------|
| Trigger | Page mount, providers data loaded |
| Logic | Partition providers: "已配置" = `active_key_count > 0 && is_enabled`; "待配置" = rest |
| Visual | Two sections with group headings. If one group is empty, render heading with "(0)" count but no placeholder. |
| Source | D-10 |

### I-02: Group Header

| Property | Value |
|----------|-------|
| Element | `<div>` with left-aligned text |
| Typography | 12px Manrope 700, `--cv4-text-muted`, uppercase tracking 0.5px |
| Spacing | `margin: 32px 0 12px` (first group: `margin-top: 0`) |
| Content | `已配置 (N)` / `待配置 (N)` |

### I-03: Provider Card Header (Modified)

| Property | Value |
|----------|-------|
| Height | 64px |
| Layout | `flex`, `align-items: center`, `gap: 12px`, `padding: 0 16px` |
| Elements (L→R) | Chevron (14px) → ProviderIcon (20px) → display_name (14px SG 700) → status + key count (12px Manrope 400 muted) → spacer → Configure button (⚙️ icon, 32×32px) |
| Hover | `background: #E5E2E108` on entire header row |
| Change from current | Replace `Bot` icon with `ProviderIcon`. Remove `Delete` button. Replace `Pencil` icon with `Settings` (Lucide). |
| Source | D-01 (no delete for presets), existing card pattern |

### I-04: Provider Description Line

| Property | Value |
|----------|-------|
| Visibility | Only when card is expanded |
| Position | Below header, above key section |
| Typography | 12px Manrope 400, `--cv4-text-muted` |
| Padding | `padding: 0 16px 8px 42px` (42px = 16px card padding + 14px chevron + 12px gap, aligns with display_name) |
| Content | Provider `description` field from API (e.g., "Google Gemini models — text, image, video, audio multimodal") |

### I-05: Configure Modal (Simplified ProviderFormModal)

| Property | Value |
|----------|-------|
| Trigger | Click configure button (⚙️) on card header |
| Title | `配置 {display_name}` (e.g., "配置 Google Gemini") |
| Width | 420px (unchanged) |
| Fields | API Key (required, text input, type="password") + Base URL (optional, text input, only shown when `sdk_type === "openai_compatible"`) |
| Base URL visibility | Show with pre-filled `default_base_url` value when `sdk_type === "openai_compatible"`. Hidden for `sdk_type === "native"`. |
| Base URL label | "Base URL (可选)" with helper text: `默认: {default_base_url}` |
| Submit label | "保存配置" |
| Cancel label | "取消" |
| Validation | API Key must be non-empty. Base URL: if provided, must start with `https://`. |
| Source | D-09, PROV-04 |

### I-06: Model List Section (New)

| Property | Value |
|----------|-------|

| Default state | Collapsed. Independent of key section expand state. |

## ProviderIcon Spec

| Provider Name | Icon | Source |
|---------------|------|--------|
| `gemini` | Google Gemini diamond logo | Inline SVG, 4-color diamond (blue/red/yellow/green) |
| `openai` | OpenAI logo | Inline SVG, single-color (`currentColor`) |
| `deepseek` | DeepSeek logo | Inline SVG, single-color (`currentColor`) |
| fallback | Lucide `Bot` | Existing icon, `--cv4-text-secondary` |

**Component API:**
```typescript
interface ProviderIconProps {
  name: string;       // provider_name key
  size?: number;      // default 20
  className?: string;
}
```

**Rendering:** `switch(name)` → inline SVG. Color inherits from `style={{ color }}` on parent or defaults to `--cv4-text-secondary`.

## State Mapping

| UI State | Condition | Visual Treatment |
|----------|-----------|-----------------|
| Loading | `isLoading === true` | 3 skeleton cards (64px height, pulse animation) — existing pattern |
| Error | `isError === true` | Centered error card with heading + body + retry button |
| Empty (all configured) | "待配置" group has 0 items | Group heading shows "(0)", no placeholder |
| Empty (none configured) | "已配置" group has 0 items | Group heading shows "(0)", "待配置" section shows all presets |
| Model list loading | Models query loading | Inline spinner (12px) next to "模型列表" header |
| Model list empty | Provider has 0 models | Single row: "暂无模型" in `--cv4-text-muted`, centered |
