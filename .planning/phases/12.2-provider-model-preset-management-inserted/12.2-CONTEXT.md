# Phase 12.2: Provider & Model Preset Management - Context

**Gathered:** 2026-04-03
**Status:** Ready for planning

<domain>
## Phase Boundary

将 Provider 管理从"用户手动创建"改为"系统预置 + 用户填写 Key/BaseURL"模式。预置常用 Model，用 ModelPricing 作为 Provider-Model 关联兼定价表（删除 AIModelProviderMapping），模型列表嵌入 Provider 卡片内展示（Dify 风格分"已配置"/"待配置"两组）。

**不包含在此 Phase：**
- Team/Personal scope 的 Provider key 配置入口（本阶段仅改 Admin system scope）
- Dify 式插件化 Provider 热插拔（需要单独 Phase）
- Pricing 页面改造（已有 Pricing CRUD 页面保持不变）
- 独立的 Admin Models 页面（模型列表嵌入 Provider 卡片）

</domain>

<decisions>
## Implementation Decisions

### 自定义 Provider 支持
- **D-01:** 仅代码实现的 Provider 可用（Gemini/OpenAI/DeepSeek）。预置 = 代码注册，无前端自定义创建入口。移除 "Add Provider" 按钮
- **D-02:** Team/Personal scope 本阶段不改——团队也只能使用预置 Provider，无入口添加自己的 key。Admin 页面仅管理 system scope

### 种子数据策略
- **D-03:** 种子数据带 `seed_version` 字段。每次启动只处理版本更新的记录——新模型自动添加，用户修改过的记录不被覆盖
- **D-04:** 每个 Provider 实现类声明自己的模型元数据（models + 默认定价 + default_base_url），种子函数集中读取各 Provider 声明统一写入 DB。新增 Provider 只需在实现类中定义元数据

### 定价展示
- **D-05:** Provider 卡片内模型列表只读展示定价，不可内联编辑。定价编辑走已有 Pricing 页面（`/admin/pricing`）
- **D-06:** 未定价的模型行显示"未定价"灰色标签

### base_url 配置
- **D-07:** base_url 是 SDK 层面的概念，与具体供应商无关。使用原生 SDK 的 Provider（Gemini → ChatGoogleGenerativeAI）不需要 base_url；使用 OpenAI 兼容 SDK 的 Provider（DeepSeek → ChatOpenAI）需要预定义 base_url
- **D-08:** 每个 Provider 实现类声明 SDK 类型和 default_base_url。`resolve_langchain_llm` 从 DB 读取 base_url（有 Admin 覆盖用覆盖，否则用默认值）。DeepSeek 的硬编码 base_url 改为从 DB 读取
- **D-09:** 前端配置弹窗根据 SDK 类型判断是否显示 base_url 字段——仅 OpenAI 兼容类 Provider 显示

### 前端分组与展示
- **D-10:** "已配置" = `active_key_count > 0` 且 `is_enabled=true`；"待配置" = 无 active key 或未启用
- **D-11:** 模型被 Admin 禁用后，Agent 推理时不可选，前端模型选择器不展示

### 数据迁移
- **D-12:** `AIModelProviderMapping` 当前无业务数据，直接删除表和模型类，无需迁移脚本
- **D-13:** `ModelPricing` 新增 `provider_config_id` + `model_config_id` FK 字段，原 `provider` / `model` 文本字段保留向后兼容，新记录同时写两套

### Claude's Discretion
- Provider icon 的具体渲染方式（SVG 组件 vs 图片 URL vs 字符串映射）
- 模型行的具体 UI 样式和间距
- 种子数据的具体 seed_version 格式和递增策略
- ModelPricing 文本字段到 FK 字段的查询切换时机

</decisions>
