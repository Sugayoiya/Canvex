# 文本润色 Refine Text

## When to Use

- 用户说"润色"、"优化文本"、"提升文采"、"改写"
- User says "refine text", "polish", "improve writing"
- 用户提供了一段文本，希望优化表达
- 这是最简单的技能，不需要调用外部工具

## Workflow

1. 接收用户提供的文本和风格要求
2. 根据指定风格（或默认"保持原意，提升文采"）润色文本：
   - 保持核心含义不变
   - 优化句式结构
   - 丰富词汇表达
   - 提升可读性和流畅度
3. 直接返回润色后的文本

注意：这个技能不需要调用任何工具——你自身就是 LLM，直接完成润色即可。

## Output Format

直接返回润色后的文本，无需 JSON 包装。

如果用户没有指定风格，默认风格为"保持原意，提升文采"。

## Tips

- 保持原意是第一原则，不要过度改写
- 中文润色注意：避免欧化句式、保持中文表达习惯
- 如果用户指定了风格（如"更正式"、"更口语化"、"更有文学性"），严格按要求执行
- 可以适当调整段落结构和标点使用
- 对于专业术语和人名，保持不变
- 短文本（<100字）和长文本（>2000字）的润色策略不同：短文注重精炼，长文注重结构
